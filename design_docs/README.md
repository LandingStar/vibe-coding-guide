# Design Notes Index

## 文档定位

本目录收纳基于 `review/` 研究结论整理出的**内部设计文档**。

它们的作用是：

- 把“对外部产品的观察”转化为“对本平台的设计方向”
- 记录当前准备吸收哪些模式、回避哪些模式
- 为后续修订 `docs/` 下的平台权威文档提供中间层依据

## 与其他目录的关系

- `review/`
  - 记录外部产品研究本身
- `design_docs/`
  - 记录从研究到本平台设计的推导
- `docs/`
  - 记录当前平台的正式权威文档

换句话说：

- `review/` 是原始材料
- `design_docs/` 是设计加工层
- `docs/` 是当前定稿层

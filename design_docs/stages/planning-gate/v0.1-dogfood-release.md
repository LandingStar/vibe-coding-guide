# Planning Gate: v0.1-dogfood Release

> 状态: **approved** — 用户已确认方案 E (MCP + Instructions 生成)  
> 背景: Phase 21 完成 (431 tests)，用户提议产出一个可自举使用的 release  
> 目的: 平台在本项目中 dogfood 使用，兼作集成测试和需求发现  
> 集成方案: **方案 E — MCP Server + Instructions 生成**  
> 方案依据: [copilot-integration-runtime-simulation.md](copilot-integration-runtime-simulation.md)

## 动机

当前平台各组件均已单独完成并通过测试，但没有统一的编排层和可执行入口。
产出 release 后可以：
1. 在本项目中实际使用，验证端到端流程
2. 发现文档/规范与实际使用之间的差距
3. 以真实使用催生新需求，而不是凭空推导

## 为什么选方案 E

本项目有 8 条强约束（C1-C8），包括"禁止终止对话"、"Phase 完成后自动推进"等。
分析表明（见运行时模拟文档）：
- 方案 A/C/D 的约束是 advisory（建议性），LLM 可不遵守
- 方案 E 的 MCP 约束是 structural（结构性），tool 返回 BLOCK/ALLOW 信号
- MCP tool 列表不受上下文压缩影响，instructions 层提供保底

## Release Scope: v0.1-dogfood

### 切片 1: Pipeline Orchestrator + CLI（关键路径，所有方案共用基础）

**目标**: 一条命令跑通完整治理链

新增文件:
- `src/workflow/pipeline.py` — Pipeline 编排类
  - `Pipeline(pack_dirs, project_root, dry_run=True, audit=True)`
  - `.process(input_text) → PipelineResult`
  - 内部串联: load packs → build context → resolve rules → build_envelope → execute → writeback
- `src/__main__.py` — CLI 入口
  - `python -m src process "用户输入"` — 运行治理链
  - `python -m src info` — 显示已加载 pack 信息
  - `python -m src validate` — 校验项目配置

验收标准:
- [ ] `python -m src process "请修改 checklist"` 输出完整 envelope + execution result
- [ ] dry-run 不产生任何文件副作用
- [ ] non-dry-run 可实际执行 writeback
- [ ] 审计日志自动生成

### 切片 2: MCP Server 骨架

**目标**: 将 Pipeline 暴露为 MCP tools，供 Copilot 结构性调用

新增文件:
- `src/mcp/server.py` — MCP Server 主进程
  - 使用 Python MCP SDK（`mcp` package）
  - 常驻进程，保持状态
- `src/mcp/tools.py` — MCP tool 定义
  - `governance_decide(input: str)` → PDP/PEP 结果 + 约束检查
  - `check_constraints()` → 当前约束违反列表 + files_to_reread
  - `get_next_action()` → 推荐下一步 + 文档引用
  - `writeback_notify(phase_completed: str)` → 自动推进建议

MCP Tool 行为规范:
- `governance_decide` 在以下情况返回 `BLOCK`:
  - 无 planning-gate 文档时 (C5)
  - 检测到 scope 偏移时 (C6)
  - 高影响决策未经审核时 (C7)
- `get_next_action` 始终返回:
  - `instruction`: 需要执行的动作
  - `document_refs`: 引用的文档路径 (C2)
  - `ask_user`: 是否需要向用户提问 (C1)
- `writeback_notify` 在 phase 完成时返回:
  - `auto_next.candidates`: 候选方向 + 来源 (C3)

验收标准:
- [ ] `python -m src.mcp.server` 启动 MCP server
- [ ] VS Code 可通过 MCP 配置连接
- [ ] governance_decide 返回结构化约束检查结果
- [ ] BLOCK 场景测试通过（无 planning-gate → BLOCK）

### 切片 3: Instructions Generator

**目标**: 从 pack 自动生成 copilot-instructions.md 相关段落

验收标准:
- [ ] 生成的 instructions 包含 pack 声明的所有 document_types
- [ ] 包含 pack 声明的所有约束
- [ ] 生成结果可直接粘贴到 copilot-instructions.md

# Planning Gate Prompt

请先不要直接编码。

先基于当前仓库已有文档与代码现实，生成或刷新一份窄 scope planning-gate 文档。该文档至少要明确：

- 当前问题是什么
- 本轮只做什么
- 本轮明确不做什么
- 验收与验证门
- 需要同步更新哪些文档、帮助或提示词
- 当前切片会直接复用哪些本仓库已存在的文档型成果
- 若计划使用 Pipeline / CLI / MCP / Instructions 等运行时入口，要说明它们是稳定默认路径还是 pre-release dogfood 验证入口
- 为什么这个切片可以独立收口

若本轮需要用户做方向确认、审批或范围取舍：

- 先给出 AI 自身的分析、推荐或倾向判断
- 再用 askQuestions 或明确提问继续推进
- 不要以“请选择方向”或普通总结作为终止式收尾

如果已有 active planning doc，先更新它；如果没有，再从模板创建。

# 📄 补充文档 1-A：基于速度特判的绝对时停与冻结机制

**(Absolute Time-Freeze via Speed Special Judgement)**
    

## 1. 核心问题与特判逻辑

在处理减速 Debuff 时，必须在底层数学模型上严格区分“跳过回合（Stun）”与“行动条冻结（Freeze）”。本组件允许实体的当前速度 $V \le 0$，并通过极其精简的数学特判来处理该状态，避免引入冗余的距离缓存变量或引发除以零异常。

## 1.1 进入冻结状态 ($V_{new} \le 0$)

当实体的速度发生突变，且新速度 $V_{new} \le 0$ 时，触发冻结管线：

- **距离缓存转化：** 此时不执行常规的除法运算，而是将除数强制置为 1。实体面板上的 $AV$ 变量在物理意义上直接转化为**剩余距离 ($D_{rem}$)**。
    
- **计算公式：**
    
    $$AV_{frozen} = AV_{old} \times V_{old}$$
    
- **Tick 豁免：** 在时间轴组件执行全局时间推进（Tick）时，特判并**跳过所有 $V \le 0$ 的实体**，不执行任何减法操作，实现真正的“绝对时停”。
    

## 1.2 解除冻结状态 ($V_{new} > 0$)

当实体速度恢复正数时：

- 此时其 $AV$ 变量内存储的正是被冻结前的剩余距离。直接使用该距离除以新速度，将物理量重新转化为时间。
    
- **计算公式：**
    
    $$AV_{new} = \frac{AV_{frozen}}{V_{new}}$$

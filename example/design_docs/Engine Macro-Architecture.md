# 📄 文档 0：引擎宏观架构纲领——微内核与依赖注入

**(Engine Macro-Architecture: Microkernel & Dependency Injection)**

## 1. 核心思想：微内核架构 (Microkernel Architecture)

本引擎采用极其严格的**微内核架构**。 整个框架被物理切割为两个绝对隔离的层级：

- **Core (引擎底座):** 极度纯洁、无知且稳定。它仅包含最基础的数据容器（如 `Entity` 字典）和流转枢纽（如 `EventBus`）。Core 内部**绝对不包含任何具体的业务逻辑**（不认识什么是时间轴、不认识什么是回合、不认识什么是标签 Tag）。它只负责定义规则的**接口 (Interfaces)**。
    
- **Standard Components (官方标准组件):** 引擎开箱即用提供的组件库。它们实现了 Core 定义的接口，包含了所有复杂的底层业务逻辑（如 AV 时间轴计算、AST 标签校验等）。
    

## 2. 交互协议：依赖倒置与注入 (Dependency Injection)

Core 绝不主动实例化任何具体的组件。在引擎生命周期初始化时，引擎使用者（业务开发者）必须将实例化后的组件，作为**服务 (Services)** 注入到 Core 中。

这种设计赋予了引擎极大的魔改空间：开发者可以随时拔掉官方的“标签校验组件”，插入自己手写的“Lua 脚本校验组件”或“A* 寻路网格校验组件”，而不需要修改 Core 的哪怕一行代码。

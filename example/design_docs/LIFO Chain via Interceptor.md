# 📄 子文档 3-B-1：基于拦截器的时序重塑 (LIFO 连锁机制实现)

**(Advanced Sequencing: LIFO Chain via Interceptor)**

## 1. 架构痛点：宣告与结算分离

部分策略游戏（如《游戏王》的“连锁”或《万智牌》的“堆叠”）要求后发先至的 LIFO（先进后出）结算逻辑。玩家宣告发动技能后，需进入响应窗口，随后倒序结算。这种逻辑与 Core 默认的 DFS 即时嵌套相冲突。

## 2. 标准解法：连锁结算组件 (Chain Resolution Component)

引擎使用者无需修改 Core，只需实现并注入一个继承自 `IEventInterceptor` 的组件：

1. **事件劫持：** 组件拦截玩家发出的 `SkillCastEvent`，扣留于组件内部的待处理连锁栈（Chain Stack）中，阻止其进入总线执行流。
    
2. **响应窗口：** 组件抛出系统级事件开启响应倒计时，允许对端玩家提交响应事件（如“打断”或“反击”），并继续劫持压栈。
    
3. **倒序反刍：** 当双方放弃响应，组件将内部栈自顶向下 `pop`（后发先至），重新 `push` 给 Event Bus 进行标准结算。
    
    配合 3-A 的 TOCTOU 校验管线，后发事件可直接修改先发事件的状态（如挂载“取消”标签），实现完美的无缝反制。

# Issue: 项目类型/接口依赖关系管理与耦合联动追踪

## 标题

平台需要具备对项目中类型、接口的调用/继承/联动关系的管理能力，以及在变更发生时自动钩起耦合点的联动提醒机制。

## 需求描述

### FR-1：类型与接口关系管理

管理项目中各种类型和接口的调用、继承、联动关系，包括但不限于：

- 类型之间的继承/实现关系
- 接口与实现之间的对应关系
- 函数/方法之间的调用链
- 跨模块的类型依赖图

目标：让平台能够维护一张可查询的关系图谱，辅助 agent 和用户理解模块间的结构性依赖。

### FR-2：变更联动钩子（耦合点追踪）

记录无法解耦的联动关系，在某处被更改时自动触发相关位置的检查或提醒：

- 标注哪些位置存在耦合（例如：改了 A 必须同步改 B）
- 在变更发生时，钩起（hook）所有受影响的耦合点
- 用于维护那些因设计约束而无法解耦的内容

目标：降低因遗漏联动更新导致的不一致风险，使 agent 在执行变更时能主动检查所有耦合点。
